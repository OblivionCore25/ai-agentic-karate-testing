Database Schema Ingestion

SQLAlchemy inspect() extracts tables, views, and all constraints from a live PostgreSQL instance.

Table: orders (schema: public)
Description: GOLD tier customers receive 10% discount when total > $500

Columns:
  - id: UUID          [PK, NOT NULL, DEFAULT gen_random_uuid()]
  - customer_id: UUID [NOT NULL, FK → customers.id]
  - status: VARCHAR   [NOT NULL, DEFAULT 'PENDING']
  - total_amount: NUMERIC(12,2) [NOT NULL, CHECK >= 0]
  - discount_applied: NUMERIC(12,2) [DEFAULT 0.00]

Foreign Keys:
  - customer_id → customers.id ON DELETE CASCADE

Each table becomes a searchable chunk in ChromaDB — the LLM uses this to discover test scenarios invisible to the API spec.

JDBC Database Verification

Generated features include inline JDBC steps that verify database state after API calls.

✅ Happy Path — Verify Write

# After POST /orders → 201
* def conn = DriverManager
    .getConnection(dbUrl, props)
* def stmt = conn.prepareStatement(
    "SELECT status, total_amount
     FROM orders WHERE id = ?")
* match rs.getString('status')
    == 'PENDING'
* match rs.getBigDecimal('total_amount')
    .doubleValue() == 550.0
* eval conn.close()

❌ Error Path — Verify No Write

# After FK violation → 400
* def stmt = conn.prepareStatement(
    "SELECT COUNT(*) AS cnt
     FROM orders
     WHERE customer_id = ?")
* eval stmt.setObject(1,
    UUID.fromString(
      '00000000-...'))
* match rs.getInt('cnt') == 0
* eval conn.close()
